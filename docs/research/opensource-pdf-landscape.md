# 开源 PDF 阅读器调研

## 1. iOS/macOS Swift 项目现状

### 关键发现：市场空白

原生 Swift/SwiftUI PDF 阅读器领域**极度碎片化**，无主导项目。

| 项目 | Stars | 技术栈 | 状态 | 特点 |
|------|-------|--------|------|------|
| FolioReaderKit | ~2,700 | Swift, EPUB 为主 | 零星维护 | 标注 UX 可参考 |
| UXReader-iOS | ~200-400 | ObjC/Swift, Core Graphics | 不活跃 (2020-2021) | 自定义渲染引擎 |
| Kiwix-apple | ~400+ | Swift/SwiftUI | 活跃 | 离线阅读器架构参考 |
| 各种 PDFKit wrapper | <500 | SwiftUI | 多数已废弃 | 碎片化，无一突出 |

**为什么存在空白**：
- PDFKit "够用"降低了构建替代品的动力
- 认真做 PDF（表单、高级标注）需要巨大工程量
- 商业玩家（PSPDFKit, PDF Expert）占据质量高地

### 结论：Aether Reader 有真正的机会成为第一个严肃的原生 SwiftUI PDF 阅读器

## 2. 跨平台成功项目

| 项目 | Stars | 技术栈 | 状态 | 成功原因 |
|------|-------|--------|------|----------|
| **Sioyek** | ~7,000+ | C++, OpenGL, MuPDF | 创建者已去世，社区 fork 碎片化 | 学术/研究聚焦：智能跳转、传送门视图、键盘驱动 |
| **Okular** | N/A (KDE) | C++/Qt | 持续维护 | KDE 生态集成、多格式、机构支持 |
| **Zathura** | ~2,000+ | C, GTK | 维护中 | Unix 哲学：只做一件事做好。Vim 用户最爱 |
| **SumatraPDF** | ~13,000+ | C/C++ | 活跃维护 | 极快启动、极小体积、零膨胀 |
| **MuPDF** | ~1,500+ | C | AGPL | 底层渲染引擎，被 Sioyek/Zathura 等使用 |

### 成功项目的共同特征
1. **聚焦的定位**：每个项目都选了一个细分（键盘用户/速度/研究/Linux 桌面）
2. **渲染性能优先**：功能广度让位于速度
3. **最小化 UI**：更少 chrome，更多文档
4. **键盘优先**：重度用户驱动贡献
5. **插件/可扩展架构**：社区贡献不需改核心

## 3. 商业模式

| 模式 | 代表 | 可持续性 |
|------|------|----------|
| 捐赠 | Sioyek, Zathura | ❌ 几乎不可持续 |
| 双重授权 (AGPL + 商业) | MuPDF | ✅ 最可持续 |
| 开放核心 + 付费功能 | Foxit | ✅ 可行 |
| 机构/企业授权 | Okular/KDE | ⚠️ 需组织支撑 |
| **App Store + IAP** | **Aether Reader 可选** | ✅ 探索不足的模式 |

**推荐**：GitHub 开源核心 + App Store 发布精品版 + 高级功能 IAP（AI 摘要、云同步、高级标注）

## 4. 常见陷阱

| 陷阱 | 原因 | 缓解 |
|------|------|------|
| **项目废弃** (#1) | 低估 PDF 格式复杂度 | 用 PDFKit 做渲染，专注 UX 层 |
| **性能问题** | 大文档滚动卡顿 | 分块渲染、后台预取、页面回收 |
| **标注薄弱** | 大多只有基础高亮 | 这是**差异化机会** |
| **平台限制** | PDFKit 已知 bug（选区、渲染等） | 混合方案：PDFKit 渲染 + 自定义标注 overlay |
| **范围蔓延** | 想同时做编辑器+阅读器+表单 | 选 2-3 个核心做到极致 |
| **UI/UX 忽视** | 开发者驱动项目重功能轻设计 | **Aether Reader 最大的差异化** |

## 5. Aether Reader 差异化机会

### Gap 1: 不存在现代美观的原生 SwiftUI PDF 阅读器
- 所有主流开源项目是 C++/Qt/GTK 或 UIKit 老设计
- "做 PDF 阅读器中的 Things 3" — 功能别人也有，但设计卓越

### Gap 2: AI 增强阅读
- 无开源 PDF 阅读器有 AI 集成
- 功能方向：章节摘要、段落解释、关键点提取、文档问答、语义搜索
- Core ML 本地推理保护隐私 + 可选云端 AI

### Gap 3: 继承 Sioyek 的研究工作流
- Sioyek 7K+ stars 证明需求存在，但创建者已去世，社区 fork 碎片化
- 智能跳转、传送门视图、参考文献检测 — 在原生 Apple 应用中重新实现

### Gap 4: 标注 UX 达到商业级
- 开源标注普遍很弱
- Apple Pencil 低延迟墨迹、彩色高亮+标签系统、边注、标注搜索、导出为 Markdown

### Gap 5: Apple 生态无缝集成
- 无开源项目做跨设备同步
- CloudKit/iCloud、Handoff、ShareSheet、Shortcuts — 全是原生免费能力

### Gap 6: 现代文档库管理
- 大多数阅读器一次开一个文件
- 集成书库：智能文件夹、标签、阅读状态、元数据提取

## 6. 战略定位建议

> **"为研究者和深度阅读者打造的、设计精美的、AI 增强的 PDF 阅读器，原生于 Apple 生态。"**

对标：
- 商业应用 → 开源 + 隐私优先
- 现有开源 → 卓越设计 + AI
- Sioyek 遗产 → Apple 平台上的延续和进化
- 通用 PDF 工具 → 聚焦阅读/研究工作流，不做编辑
